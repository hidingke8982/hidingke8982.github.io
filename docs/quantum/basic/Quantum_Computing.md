# Quantum Computing



## 양자 컴퓨팅이란
양자 컴퓨팅은 고전 컴퓨터가 처리하기 힘든 문제를 양자 역학의 고유한 현상인 중첩(Superposition)과 얽힘(Entanglement)을 활용해 정보 처리 및 계산에 활용하는 새로운 패러다임이다.



## 워크플로우
양자 컴퓨팅의 전체 과정은, 문제를 정의하고, 양자 알고리즘을 설계하며, 이를 실제 하드웨어에서 실행한 후 결과를 해석하는 일련의 단계로 이루어진다. 이 과정은 고전 컴퓨터와 양자 컴퓨터 사이를 오가며 진행된다.

### 1. 양자 알고리즘 설계 (고전 컴퓨터)
- 문제 식별
- 양자 알고리즘 선택/설계
- 양자 회로 표현

### 2. 회로 최적화 및 변환 (고전 컴퓨터)
- 트랜스파일 (Transpile): 사용자가 설계한 회로의 양자 게이트들을 타겟 하드웨어가 기본적으로 지원하는 게이트 세트로 변환하고, 성능을 최적화하는 과정.
    - 매핑 (Mapping): 논리 큐비트를 물리 큐비트에 할당하고, 하드웨어의 연결성 제약을 만족시키도록 회로를 재구성(SWAP 게이트를 회로에 추가)하는 과정.
        - 논리 큐비트(Logical Qubit): 사용자가 회로를 설계할 때 추상적으로 생각하는 $q_0, q_1, q_2, \dots$ 와 같은 큐비트.
        - 물리 큐비트(Physical Qubit): 실제 양자 QPU 위에 존재하는 고유한 위치와 번호를 가진 큐비트.
    - 변환 (Decomposition): 사용자가 복잡하거나 고수준의 게이트(예: $R_x(\theta), R_y(\theta), Toffoli$)를 사용했다면, 트랜스파일러는 이를 하드웨어가 직접 실행할 수 있는 기본 게이트의 조합(예: $\{H, R_z, CNOT\}$)으로 분해한다.
        - 논리 게이트(Logical Gate): 양자 회로를 설계할 때 사용자(프로그래머)가 알고리즘의 논리를 표현하기 위해 사용하는 모든 양자 게이트.
        - 기본 게이트(Native Gate): 특정 하드웨어가 물리적으로 직접 구현하고 제어할 수 있는 게이트. 이는 하드웨어의 물리적 특성(초전도, 이온 트랩, 등)에 의해 결정된다.
    - 최적화 (Optimization): 게이트의 총 수(회로 깊이)를 줄여서 오류율을 최소화한다. 양자 게이트는 실행될 때마다 노이즈를 발생시키기 때문에, 게이트가 적을수록 전체 계산의 정확도가 높아진다.
- 컴파일 (Compile): 트랜스파일링을 통해 변환 및 최적화된 양자 게이트 시퀀스를 실제 하드웨어를 제어하는 물리적 펄스 또는 마이크로파 신호의 시퀀스로 최종 변환하는 과정.
    - 게이트-펄스 변환: 각 기본 양자 게이트(예: $X$ 게이트, $C_X$ 게이트)는 특정 주파수, 진폭, 지속 시간을 가진 마이크로파 펄스로 변환된다. 이 펄스가 큐비트에 인가되어 원하는 양자 상태 변화(회전)를 유도한다.
    - 시퀀스 생성: 컴파일러는 이 펄스들을 시간 순서대로 정확하게 배열하여 펄스 시퀀스를 생성한다. 이는 하드웨어의 제어 시스템이 이해하고 실행할 수 있는 저수준의 명령어다.

### 3. 하드웨어 실행 (양자 컴퓨터)
- 초기화: 모든 큐비트의 상태를 기준 상태($\vert 0 \rangle$)로 초기화합니다.
- 펄스 인가: 생성된 펄스 시퀀스를 큐비트에 인가합니다. 이 펄스는 큐비트 상태를 중첩시키고, 얽히게 하며, 원하는 계산을 수행합니다.
- 측정 (Measurement): 계산이 완료되면, 회로의 끝에 있는 측정 게이트가 큐비트의 양자 상태를 **고전적인 비트 값(0 또는 1)**으로 붕괴시키고 이 결과를 기록합니다.
- 반복 실행 (Shots): 양자 계산은 확률적이므로, 통계적으로 유의미한 확률 분포를 얻기 위해 이 과정을 수백에서 수천 번 반복하여 수행한다다.

### 4. 결과 수집 및 처리 (고전 컴퓨터)
- 결과 수집: 하드웨어로부터 반복 실행을 통해 얻어진 **출력 문자열(비트열)**과 그 **관측 빈도(확률 분포)**를 수신합니다.
- 후처리: 수집된 확률 분포를 분석하여 원래 문제의 해답을 찾습니다.
    - 양자 상태 단층 촬영(Quantum State Tomography): (필요한 경우) 확률 분포를 이용하여 시스템의 **밀도 행렬($\hat{\rho}$)**을 재구성하여 큐비트의 최종 상태를 확인합니다.
    - 오류 완화 (Error Mitigation): 하드웨어 노이즈로 인한 오류를 통계적 기법을 사용해 보정하고 최종 해답의 정확도를 높입니다.
- 응답 도출: 후처리된 결과를 기반으로 원래 문제에 대한 최종 해답을 도출합니다.

### 5. 피드백 및 반복
- 검증: 도출된 해답이 정확한지 검증합니다.
- 피드백: 결과가 원하는 정확도에 미치지 못하면, 1단계(알고리즘), 2단계(최적화/매핑), 3단계(하드웨어 설정) 중 문제가 된 부분을 찾아 수정하고 워크플로우를 반복합니다.
